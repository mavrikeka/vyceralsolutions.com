The Exploration

Consulting frameworks delivered exclusively through consultant time hit capacity ceilings—can productizing a single element (JTBD writing) into a gamified app create self-service value?

  • Capacity constraints: Teaching JTBD writing skills consumes consultant time that could focus on higher-value strategic work
  • Reinforcement gap: Clients need ongoing practice applying learnings to new hiring/succession decisions post-engagement
  • Scalability question: Can interactive learning replace traditional delivery, or does the framework require human expertise?

The Learning: Beta MVPs test productization hypotheses before major investment—validating whether frameworks can scale through software or need consultant delivery.

The MVP

Full-stack mobile/web application with three-mode gamified learning (Learn-Build-Critique) and AI-powered feedback to test framework productization viability.

MVP Deliverables:

  • iOS & web deployment — Single codebase (Expo) published to AppStore + web for accessibility testing
  • Three-mode learning — Learn Mode (10 Fortune 500 examples), Build Mode (guided creation), Critique Mode (AI feedback)
  • Offline-first design — Local storage, no sign-in required, works without internet for learn mode
  • Production-ready infrastructure — Railway backend, Claude Code API, progress tracking, support site

Strategic Approach:

  • Fast time-to-market (weeks not months) using Expo + Railway + Claude Code for rapid full-stack development
  • Beta pilot approach tests hypotheses (gamified learning, self-service viability, AI sufficiency) before scaling investment
  • Single-element productization focuses on ONE teachable skill (JTBD writing) vs. attempting to productize entire framework

The Learning: Framework productization requires hypothesis validation through working MVPs. Beta pilots reveal whether self-service works, AI feedback suffices, and whether isolated elements create value—informing scale/pivot decisions.
