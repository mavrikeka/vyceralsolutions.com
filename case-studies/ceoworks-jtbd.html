The Exploration

CEOWorks has a comprehensive Talent To Value consulting process for matching critical talent to strategic work. One key element—writing Jobs-to-be-Done (JTBD) statements—is particularly challenging for clients to master. Could this single skill be taught through a gamified mobile/web app?

Hypotheses Being Tested:

  • Gamified Learning Works: Can interactive learn-build-critique modes teach JTBD writing skills more effectively than traditional consulting delivery?
  • Self-Service Is Viable: Will clients use a standalone app to practice and improve their JTBD writing without consultant hand-holding?
  • AI Feedback Suffices: Is AI-powered critique sufficient to guide improvement, or do clients need human expert feedback?
  • Framework Productization Potential: Does productizing ONE element of the Talent To Value process create value, or does the full framework need to remain consultant-delivered?
  • Ongoing Client Touchpoints: Will an app create post-engagement relationships, enabling clients to continue applying learnings to new hiring/succession decisions?
  • Monetization Viability: Is there a market for subscription/licensing models, or do clients expect this as part of consulting engagements?

Project Status: This is a beta MVP designed to test these hypotheses in the market. Early pilot users will provide data on whether gamified framework productization is a viable path forward.

The MVP

Vyceral Solutions designed and built JTBD Mastery, a beta mobile/web application that gamifies learning one element of the Talent To Value process—teaching users to write better JTBD statements through interactive practice and AI-powered feedback.

MVP Design Approach:

1. Three-Mode Learning Architecture
  • Learn Mode: 10 real-world examples from Fortune 500 companies (VP Operations, CTO, CMO) showing excellent vs. vague JTBD statements—teaching by contrast
  • Build Mode: Interactive guided creation of JTBD statements with step-by-step prompts for WHAT (specific work), HOW MUCH (measurable outcomes), WHEN (concrete deadlines)
  • Critique Mode: AI-powered analysis of user-submitted JTBD statements with specific feedback on measurability, missing elements, and improvement suggestions
  • Progressive learning path: study examples → practice building → get AI feedback on your work
2. AI-Powered Feedback System
  • Backend API powered by Claude Code analyzing JTBD statements for quality
  • Evaluates whether outcomes are specific and measurable (not generic)
  • Identifies missing elements (work definition, metrics, deadlines)
  • Provides concrete suggestions for strengthening statements
  • Suggests AI-generated refinements while building statements in real-time
3. Mobile-First Experience
  • Expo framework: Single codebase deployed to both iOS (AppStore) and web
  • No sign-in required: Immediate access—download and start learning without registration friction
  • Local data storage: All user progress, created statements, and work history stored on device for privacy
  • Offline access: Learn mode examples available without internet connection
  • Clean, distraction-free interface focused entirely on learning
4. Full-Stack Application Architecture
  • Frontend: React Native with Expo for mobile/web—beautiful UI/UX with native feel
  • Backend: Railway-hosted API for AI critique and suggestions
  • Development: Claude Code accelerated full-stack development with AI-assisted coding
  • Distribution: AppStore submission for iOS with web version for broader access
  • Complete product: authentication, progress tracking, history, data management
5. Framework Productization Strategy
  • Core JTBD framework translated into interactive learning experience
  • Real business scenarios (hiring, succession, key initiatives) embedded as practice contexts
  • Quality examples demonstrate "If you can find it on Google, it's not a JTBD"
  • Progress tracking shows examples studied, statements created, critiques received
  • Support infrastructure: documentation site, contact email, feedback loops

MVP Capabilities - What Can Now Be Tested:

  • Gamified learning validated: Working three-mode system (Learn-Build-Critique) ready for user testing
  • Self-service experience built: Users can practice JTBD writing independently without consultant involvement—hypothesis testable
  • Multi-platform deployment: Mobile (iOS AppStore) and web versions enable testing accessibility preferences
  • AI feedback functional: Critique mode operational—can now test whether AI guidance suffices or human expertise needed
  • Monetization groundwork: Foundation exists to test subscription/licensing models vs. free tier strategies
  • Post-engagement tool available: Clients can test using app repeatedly for new hiring/succession decisions after consulting ends

What's Next: Beta pilot users will provide data on whether gamified learning works for JTBD skills, if clients engage with self-service tools, and whether this single-element productization creates value—informing decisions about scaling or pivoting the approach.

Development Approach

From framework to functioning app in weeks, not months

Phase 1: Framework Translation & UX Design

  • Collaborated with CEOWorks to extract core JTBD methodology
  • Identified 10 best Fortune 500 examples for Learn mode
  • Designed three-mode learning path (Learn → Build → Critique)
  • Created mobile-first interface mockups and user flows
  • Defined AI critique logic and feedback types

Phase 2: Full-Stack Development

  • Built React Native frontend with Expo framework
  • Implemented local data storage for offline functionality
  • Developed Railway-hosted backend API for AI features
  • Integrated Claude Code for statement analysis and suggestions
  • Created progress tracking and work history features

Phase 3: Testing & Refinement

  • User testing with CEOWorks clients and consultants
  • Refined AI feedback quality and suggestion accuracy
  • Optimized mobile performance and offline behavior
  • Polished UI/UX based on feedback
  • Validated learning path effectiveness

Phase 4: Launch & Distribution

  • AppStore submission and approval process
  • Web deployment for broader accessibility
  • Created support documentation site
  • Established feedback channels and support infrastructure
  • Prepared marketing materials and launch communications
